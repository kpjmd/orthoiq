Intelligence Card System Overview

Key Innovation: The first medical AI system where specialist agents stake tokens on outcome predictions, creating economic incentives for accuracy and transparent proof of quality.

What Makes This Unique

  • Visible Agent Stakes: Each specialist puts "skin in the game" with token predictions, showing confidence levels
  • Consensus Visualization: Inter-agent agreement percentage shows how aligned specialists are
  • Outcome Tracking: Users validate predictions over time, creating learning loop for agents
  • Achievement-Based Rarity: Higher tiers earned through quality (all agents + high consensus + MD verification)
  • Progressive Utility: QR code unlocks expanded features, tracking, and premium access

Rarity Tier System

  • STANDARD (60%): 1-3 agents, <80% consensus - Blue theme, basic tracking
  • COMPLETE (25%): 4-5 agents, 80-89% consensus - Teal/gold theme, full specialist insights
  • VERIFIED (10%): 5 agents, 90%+ consensus, MD approved - Gold theme with holographic effects, Research Agent access
  • EXCEPTIONAL (5%): 5 agents, 90%+ consensus, MD approved, validated outcome - Rainbow holographic, Wearable integration + priority support

User Flow & Retention

1. Consultation → Agents analyze case and stake tokens on predictions
2. User Feedback Modal → Initial validation, card unlocks
3. Receive Intelligence Card → See agent stakes, predictions, consensus
4. QR Scan → Access tracking page with milestone reminders
5. Outcome Validation → Report results at 2 weeks, 4 weeks, etc.
6. Agent Learning → Predictions resolved, tokens distributed, models improve
7. Tier Progression → Unlock premium features for validated outcomes

Generative Border Patterns

Each specialist agent has a signature visual pattern that appears in the border:

  • Triage Agent: Grid structure (coordination framework)
  • Pain Whisperer: Flowing waves (pain patterns over time)
  • Movement Detective: Angular diagonals (biomechanical forces)
  • Strength Sage: Concentric circles (progressive loading)
  • Mind Mender: Organic fractals (psychological complexity)

Pattern Intensity: Higher token stakes = more prominent patterns
Consensus Effect: High consensus (90%+) = harmonious blending; Low consensus (<70%) = contrasting/chaotic

Data Displayed (Front Face)

  • Agent Panel: Each specialist with token stake amount
  • Consensus Meter: Visual gauge of inter-agent agreement %
  • Total Stake: Sum of all agent token predictions
  • Primary Prediction: Highest-confidence/highest-stake prediction highlighted
  • Verification Status: User feedback ✓, MD review status, validation status
  • QR Link: To expanded tracking and milestone reminders
  • Provenance: Case ID, timestamp, blockchain proof (Base), MD verified badge, evidence grade

QR Code Expansion (Tracking Page)

When user scans QR code, they access:

  • Full Specialist Breakdowns: 2-3 paragraphs from each participating agent
  • All Predictions with Timelines: Pain predictions (2 weeks), functional restoration (4 weeks), etc.
  • Milestone Reminders: "Remind me in 2 weeks to validate Pain Whisperer's prediction"
  • Progress Input: Report current pain level, functional status, outcomes
  • Community Comparison: "Your recovery is in the 85th percentile vs similar cases"
  • Agent Performance: See which agents were most accurate on your case
  • Premium Unlocks: Research Agent access for Verified tier, Wearable integration for Exceptional tier

Next Steps for Implementation

For Claude Code to implement this system:

  • 1. Create SVG Card Component: Build dynamic Intelligence Card using consultation data from rawConsultationData
  • 2. Map Agent Stakes: Extract token predictions from prediction market system (already implemented in backend)

  • 4. Determine Rarity Tier: Based on # of specialists, consensus %, MD review status, validation status
  • 5. Generate Border Patterns: SVG patterns based on participating specialists and their stake amounts
  • 6. Create QR Code: Link to /track/[caseId] route for outcome validation
  • 7. Build Tracking Page: Display all predictions with milestone reminders and outcome input forms
  • 8. Implement Reminder System: Farcaster/Base app notifications at prediction milestones (2 weeks, 4 weeks)
